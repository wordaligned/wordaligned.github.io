Think Again

My thinking has changed since I first wrote this article: although I believe it remains a valid presentation of Python's functional programming capabilities, I cannot whole-heartedly recommend the final script as a solution to the original problem – namely, of syntax highlighting C++ and Python source code.

The script makes the classic mistake of confusing document structure with document presentation. That is, it simply embeds the markup tags directly: despite the fact that we identified keywords, comments etc, this structural information has been lost by the time the HTML is output. It would have been better to separate structure from presentation using cascading style sheets. (See, for example, the style sheet used for this article.)

The second mistake is that it re-invents a tool which probably already exists somewhere out there. In fact there are several such tools which are more powerful, more flexible, freely available and suitably licensed.

The third mistake is that, although the script is fine at processing blocks of source code, it doesn't scale very well to processing articles which embed such blocks, which is what I really wanted to do: for that job, regexes meet their match, and a parser is required.

A superior documentation toolchain might:

  1. Be a freely available open source solution – allowing me to modify it if required
  2. Accept as input a simple text format
  3. Parse this input, converting it into a standard and open format for structured technical documentation – such as DocBook.
  4. Which would then allow me to convert from DocBook into any number of formats (HTML, PDF, RTF etc) by applying the right XSL stylesheets

Whilst researching another article I chanced upon just such a tool. QuickBook is a Spirit application and can now be found in the boost/tools subdirectory of the Boost CVS repository. It allows you to generate BoostBook documentation from a WikiWiki style plain text source file.

Here is the plain text source for this article: fp_python.qbk

QuickBook is distributed as source code under the Boost software license. I helped adapt Quickbook to recognise Python source code.
