Introduction

An earlier article described the development of a Python script to automate the conversion of C++ source code into an HTML marked-up form. The finished script was far from perfect:

  • The markup scheme was hard-coded into the script: clients could not supply their own scheme.
  • The script bundled data sets together with low-level regex and html markup utilities to provide the high-level functionality. It would be easier to maintain and (re-)use if properly partitioned.
  • No unit tests accompanied the script.

This article addresses these flaws by continuing the development of the script in a test-driven manner. Unit tests are added using two of Python's test frameworks:

  • unittest, also known as PyUnit, which is Python's standard unit testing framework.
  • doctest, which verifies docstrings represent reality.
